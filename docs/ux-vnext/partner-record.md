# Partner record (CCX-502)

The additive Partner record composes the merged Partner stage and Partner activity projections. It keeps the exact Partner source identity and exposes Overview, Activity, Outreach, and Files tabs without creating a second Partner model.

The header keeps name, founder-facing stage, health, owner, and next action visible together. The dominant action completes the current next action when one exists. Log activity is a narrow internal evidence write. Create outreach and Add file open existing safe creation workflows and do not send, enroll, approve, schedule, upload, or share merely by opening.

Sensitive contacts and relationship notes require the repository's existing `read_sensitive` capability. Other actors receive an unavailable state rather than the content. Normal display omits internal lifecycle, engine, storage, provider, and audit terminology.

Requested Integration endpoints are exact Partner-scoped routes for logging activity and completing a next action. Both validate authorization, Partner visibility, request identity, and input. They write one Partner plus bounded activity/audit evidence, preserve the internal stage byte-for-byte, and perform zero external actions.
