# Hardening Run Notes (2026-07-08)

One lesson per entry. Corrections and confirmed approaches alike. Nothing here duplicates the run doc or repo docs.

- **Source-guard slice markers must pin the route form, not the path string.** `"/api/google/callback"` appears in a helper at preview-server.mjs:8142 with the condition order reversed, so a sliceBetween starting on the bare path captured 29k unrelated lines and false-failed. Start markers on `if (url.pathname === "..."` match only the route.
- **The store's own convenience methods are write sites too.** updateSocialAccount, addPublishEvent, and updateSettings are full-state read-modify-writes inside storage.mjs; a route-level grep for store.writeState misses them. Tier sweeps must include store methods and classify by their most exposed caller (updateSocialAccount was bot-reachable via the public OAuth callback).
- **The PR #30 "~60 sites" inventory undercounted.** Fresh grep this session: 93 writeState sites in preview-server.mjs plus 5 in modules (2x heartbeat.mjs, 3x reactivation CLI scripts). Re-derive counts, never reuse them.
- **Render deploys are not automatable from this environment.** No RENDER_* credential, no CLI, autoDeploy false. Promote stays a Roger dashboard click; post-deploy verification can be armed as a prod-commit watcher instead.
- **Deploy timing matters while the campaign sends.** Prod restarts kill mid-tick sends (the 12:00Z incident shape). Promotes should land between :10 and :55 UTC, never around the top of the hour, until the send window closes at 21:00 UTC.
- **Regex-matched routes hide from pathname greps.** The generic `/api/oauth/:platform/callback` handler (matched via `url.pathname.match(...)`) contained a second PUBLIC google full-state write that the tier-1 `url.pathname === "/api/..."` grep could not see. Route sweeps must also grep `pathname.match(` sites.
